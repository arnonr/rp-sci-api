#Sync schema without data lost
- npx prisma db push
- npx prisma db pull
--------------------------------------------
prisma db push เป็นคำสั่งของ Prisma ที่ใช้สำหรับสร้างหรืออัพเดทสคีมา (schema) ของฐานข้อมูลในฐานข้อมูลจริง ให้สอดคล้องกับสคีมาที่ถูกกำหนดไว้ในไฟล์ schema.prisma ของโปรเจกต์

การทำงานของ prisma db push คือ:

Prisma จะอ่านสคีมาจากไฟล์ schema.prisma
เปรียบเทียบสคีมากับสคีมาของฐานข้อมูลจริงที่เชื่อมต่ออยู่
สร้างหรืออัพเดทสคีมาในฐานข้อมูลจริง ให้ตรงกับสคีมาใน schema.prisma
โดยปกติแล้ว เราจะใช้คำสั่งนี้เมื่อ:

เริ่มโปรเจกต์ใหม่ เพื่อสร้างสคีมาฐานข้อมูลครั้งแรก
เปลี่ยนแปลงสคีมาใน schema.prisma แล้วต้องการปรับปรุงฐานข้อมูลจริงให้สอดคล้อง
prisma db push จะช่วยให้สามารถจัดการสคีมาฐานข้อมูลได้อย่างสะดวกผ่าน schema.prisma แทนที่จะต้องไปจัดการฐานข้อมูลจริงโดยตรง ซึ่งช่วยลดความผิดพลาดและเพิ่มประสิทธิภาพได้

แต่ควรใช้คำสั่งนี้ด้วยความระมัดระวัง เนื่องจากการเปลี่ยนแปลงสคีมาอาจส่งผลให้เสียข้อมูลได้ หากมีการเปลี่ยนแปลงที่ไม่สามารถย้อนกลับได้
--------------------------------------------
prisma db pull เป็นคำสั่งใน Prisma ที่ใช้สำหรับนำเข้าสคีมาของฐานข้อมูล (database schema) จากฐานข้อมูลจริงมาไว้ในไฟล์ Prisma schema ของโปรเจกต์

โดยปกติเราจะสร้างสคีมาใน Prisma schema file ก่อน จากนั้น Prisma จะสร้างฐานข้อมูลตามสคีมาที่เรากำหนด แต่ในบางครั้งเราอาจมีฐานข้อมูลที่มีอยู่แล้ว และต้องการนำเข้าสคีมาของฐานข้อมูลนั้นมาไว้ใน Prisma schema กรณีนี้เราจะใช้คำสั่ง prisma db pull

การใช้งานคือ ให้ระบุการเชื่อมต่อไปยังฐานข้อมูลจริงใน Prisma schema ก่อน จากนั้นรันคำสั่ง prisma db pull โดย Prisma จะวิเคราะห์สคีมาของฐานข้อมูล และสร้างไฟล์ Prisma schema ใหม่ที่สอดคล้องกับฐานข้อมูลนั้น

คำสั่งนี้มีประโยชน์เมื่อเราต้องการนำฐานข้อมูลที่มีอยู่แล้วมาใช้งานกับ Prisma โดยไม่ต้องสร้างสคีมาใหม่ทั้งหมด ช่วยประหยัดเวลาและลดความผิดพลาดได้
